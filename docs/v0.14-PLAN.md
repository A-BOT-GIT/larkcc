# larkcc v0.14 实施计划

> 三个核心功能：状态命令 + 卡片快捷按钮 + 会话历史管理

---

## 共享基础设施（先做，三个功能都需要）

### 0.1 新增使用统计模块 `src/usage-stats.ts`

> 只记录原始使用数据（tokens、工具次数、时长），**不计算费用**。费用由用户自行根据 token 数和模型定价换算。

数据来源：在 `claude.ts:processResultEvent` 已经能拿到 `event.modelUsage`（含 `inputTokens`、`outputTokens`、`cacheReadInputTokens`），加上 `toolCount`、`elapsedSeconds`、`session_id`、模型名。

存储格式：`~/.larkcc/usage-{profile}.jsonl`，每次对话追加一行：

```json
{
  "ts": 1778750000,
  "sessionId": "abc",
  "model": "claude-sonnet-4-6",
  "inputTokens": 1234,
  "outputTokens": 567,
  "cacheReadTokens": 8910,
  "cacheCreationTokens": 0,
  "toolCount": 3,
  "elapsedSec": 12.3
}
```

提供方法：

- `recordUsage(profile, entry)` —— 追加一条
- `loadUsage(profile, since?, until?)` —— 按时间范围读取
- `aggregateUsage(entries)` —— 按模型分组求和（输入 token、输出 token、缓存 token、对话次数、工具次数、累计耗时）

接入点：在 `processResultEvent` 末尾调用 `recordUsage`，零侵入。

### 0.2 新增会话历史模块 `src/session-history.ts`

存储格式：`~/.larkcc/sessions-{profile}.jsonl`，每个 session 一行：

```json
{
  "sessionId": "abc",
  "createdAt": 1778750000,
  "lastUsedAt": 1778750100,
  "firstPrompt": "帮我...",
  "msgCount": 5,
  "lastModel": "claude-sonnet-4-6"
}
```

提供方法：

- `recordSession(profile, sessionId, prompt)` —— 新会话首次记录
- `touchSession(profile, sessionId)` —— 更新 `lastUsedAt` + `msgCount`
- `listSessions(profile, limit=10)` —— 按 `lastUsedAt` 倒序
- `deleteSession(profile, sessionId)` —— 软删除

接入点：在 `processResultEvent` 中根据 `session_id` 是新会话还是续接，调用对应方法。

## 功能 1：`/status` + `/usage` 命令

> 命名调整：原计划的 `/cost` 改为 `/usage`，因为不再算钱。

走 `BUILTIN_EXEC` 路径，但因为需要返回卡片不是纯文本，新增分类 `BUILTIN_CARD_CMD`，让 `parseCommand` 返回新类型 `card`，由 `message-handler.ts` 直接发卡片消息。

## 功能 2：卡片快捷按钮

### 2.1 设计

在 `buildFinalCard` 末尾追加一行按钮，按当前状态决定显示哪些：

| 按钮 | 条件 | 行为 |
|------|------|------|
| 🔄 重试 | 始终显示 | 重新发送上次的 prompt |
| ➡️ 继续 | 始终显示 | 发送 "继续" 给 Claude |
| 📋 复制 | 始终显示 | 触发飞书原生复制（用 callback 回发文本） |
| 🆕 新会话 | 始终显示 | 清除当前 session_id |

### 2.2 飞书卡片按钮 + 回调机制

飞书卡片按钮点击会触发 `card.action.trigger_v1` 事件，包含：

- `operator.open_id` —— 谁点的
- `action.value` —— 按钮的自定义 payload（用它区分操作类型）
- `context.open_message_id` —— 卡片所在消息 ID
- `context.open_chat_id` —— 卡片所在群 ID

需要做：

1. **构建按钮**：在 `card/elements.ts` 加 `button(text, value, type)` 函数，在 `card/containers.ts` 加 `buttonRow(buttons)` 函数。
2. **注册按钮事件**：在 `app.ts` 的 `eventDispatcher.register` 中加 `card.action.trigger_v1` handler。
3. **回调路由**：新文件 `src/card-action-handler.ts`，根据 `action.value.type` 分发到对应处理函数：
    - `retry`：取上一条 prompt，重新触发 `runAgent`
    - `continue`：发送 "继续" 触发 `runAgent`
    - `new_session`：调 `clearSession`，回发 "✅ 新会话已开启"
    - `copy`：原文回发为 text 消息
4. **状态依赖**：要存"上一条 prompt"，在 `message-handler.ts` 加 `lastPrompt` 变量（按 chatId 维度）。

### 2.3 改动点汇总

| 文件 | 改动 |
|------|------|
| `src/card/elements.ts` | 加 `button`、`actionPayload` |
| `src/card/containers.ts` | 加 `buttonRow` 容器（用 columnSet 包多个 button） |
| `src/client/cardkit.ts` | `buildFinalCard` 追加按钮行 |
| `src/card-action-handler.ts` | 新建 |
| `src/app.ts` | 注册 `card.action.trigger_v1` 事件 |
| `src/message-handler.ts` | 暴露 `lastPrompt` 给 action handler；提取 `runAgentForChat` 复用 |

## 功能 3：会话历史管理

### 3.1 命令设计

| 命令 | 行为 |
|------|------|
| `/sessions` 或 `/ss` | 列出最近 10 个会话（卡片，含每个的 resume 按钮） |
| `/sessions resume <id>` 或 `/ss r <id>` | 切到指定 session（id 支持前 6 位短匹配） |
| `/sessions new` 或 `/ss n` | 开新会话（= 清除当前 session_id） |
| `/sessions delete <id>` 或 `/ss d <id>` | 从历史中移除某会话 |

### 3.2 会话列表卡片

```
┌──────────────────────────────────┐
│ 🗂 最近会话                       │
├──────────────────────────────────┤
│ ⭐ abc123 · 当前 · 5 条           │
│    "帮我重构这个函数..."           │
│    [当前]                         │
├──────────────────────────────────┤
│ def456 · 2h 前 · 12 条            │
│    "分析一下这个错误日志..."       │
│    [▶️ 继续此会话] [🗑 删除]       │
├──────────────────────────────────┤
│ ghi789 · 昨天 · 3 条               │
│    "添加一个 /status 命令..."      │
│    [▶️ 继续此会话] [🗑 删除]       │
└──────────────────────────────────┘
```

每个会话项含两个按钮，点击触发 action（功能 2 的基础设施）。

### 3.3 切换会话的实现

`setSession(sessionId)` 改成会同时记录到 `session-history.ts`。在 `claude.ts:runAgent` 起手就调 `getSession()`，所以无需改 runAgent 本身——只要 `/ss r <id>` 命令把 session.ts 的当前 sessionId 改掉就生效。

### 3.4 改动点汇总

| 文件 | 改动 |
|------|------|
| `src/session-history.ts` | 新建 |
| `src/card/sessions-card.ts` | 新建 |
| `src/commands.ts` | 注册 `/sessions` 及子命令 |
| `src/session.ts` | `setSession` 同步调 `recordSession`/`touchSession` |

## 风险与依赖

1. **飞书 API 限制**：`card.action.trigger_v1` 事件需要应用有「订阅卡片回调事件」权限，需要确认你的飞书应用 (`cli_a93bdd596f38dcc7`) 已开通。如果没开，需要先去开发者后台配。
2. **数据兼容**：现有 `~/.larkcc/state.json` 不动，新增的两个 jsonl 文件独立，老用户升级零迁移。
3. **不计算费用**：本版本只记录 raw token 数据，提供 `/usage` 查询累计使用量。如果未来要算钱，可以另写一个独立的 `cost-calc.ts` 脚本读取 jsonl 文件离线计算。

## 推荐的开关

每个功能加一个 config 开关，默认开启：

```yaml
features:
  status_command: true
  usage_tracking: true
  card_buttons: true
  session_history: true
```

这样以后不喜欢某个功能可以单独关掉。
